## 功能特性

### 学生管理
- 学生列表展示，包含学号、姓名、当前操行分
- 学生头像自动生成（emoji随机）
- 根据操行分显示不同颜色卡片（正常/警告/危险）

### 操行分管理
- **单个学生操作：** 加分、减分，支持自定义理由
- **小数分支持：** 自定义加减分弹窗支持 0.5 分粒度（如 0.5、1.5、2.5）
- **快速操作：** +1、+3、-1、-4 快捷按钮（整数操作）
- **全班统一加分：** 批量为所有学生加分（整数操作）
- **操作理由：** 支持预设理由（打扫卫生、班委、优秀寝室、讲话、床铺不整齐、作业未完成等）
- **预设标签联动：** 点击「优秀寝室」自动填 +3、「打扫卫生」自动填 +2，并切换为加分模式

### 数据查询
- 搜索功能（支持姓名、学号精准搜索）
- 排序功能（按学号、按分数高低）
- 历史记录查询（查看单个学生所有加减分历史）
- **历史记录删除并撤销：** 删除某条记录时自动回滚对应分数变化

### 数据导出
- 导出为Word文档（`.doc`格式）
- 包含所有学生信息、当前分数、历史记录
- 导出时间和班级信息

### 数据持久化
- 使用localStorage存储数据
- 刷新页面数据不丢失
- 支持一键重置数据

### 阶段六：Supabase 云端数据同步
**时间：** 2026-06-19  
**完成功能：**
1. ✅ 集成 Supabase 后端服务（免费 PostgreSQL 数据库）
2. ✅ 双写策略：localStorage + Supabase 同时存储，互为备份
3. ✅ CDN 不可用时自动回退纯本地存储模式（带 5 秒超时）
4. ✅ 实时监听 Supabase 数据变更，其他设备修改后自动刷新
5. ✅ 重置数据时同步清除 Supabase 云端数据
6. ✅ 多设备（电脑/手机）访问同一 HTML 文件即可共享数据

**技术要点：**
- Supabase 项目 URL：`https://yaolwvcnwbhuagehhgxd.supabase.co`
- 数据存储在 `sync_data` 表，以 JSONB 格式存放完整学生数据
- 变量名使用 `sb` 避免与 Supabase SDK 内置 `supabase` 变量冲突
- 读取策略：先同步读 localStorage 立即渲染 → 后台从 Supabase 更新（5s 超时）
- 写入策略：先写 localStorage → 后台异步 upsert 到 Supabase

### 阶段十：数据安全加固
**时间：** 2026-06-19  
**完成功能：**
1. ✅ 修复 Supabase 数据被误重置的严重 Bug
   - 问题根因：某设备 Supabase 连不上 + localStorage 为空时，代码自动初始化默认数据（全部 80 分），然后 `saveData()` 将默认数据写入 Supabase，覆盖所有真实数据
   - 修复方案：新增 `supabaseReachable` 标记，仅在 Supabase 连通且确认无数据时才初始化默认值
   - Supabase 不可达 + localStorage 为空 → 页面显示空，不写入任何数据
2. ✅ 管理员切换按钮样式优化
   - 从 header 移至控制面板「全班统一加分」上方，样式与其他按钮统一
   - 访客模式红色渐变、管理员模式绿色渐变
